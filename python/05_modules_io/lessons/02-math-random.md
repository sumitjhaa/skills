# 🎲 Math & Random
<!-- ⏱️ 10 min read | 🟢 Easy | 🧠 Core -->

**What You'll Learn:** Use `math` for numeric operations and `random` for simulation, sampling, and probability.

> 💡 **TL;DR — The whole point:** `math` gives you scientific computing basics; `random` lets you simulate chance — dice rolls, card draws, Monte Carlo estimates.

## 🔗 Why This Matters
The import system lets you load modules. Now use `math` and `random` — two of the most commonly used standard library modules — for game mechanics, statistical sampling, and risk analysis.

## The Concept
The `math` module provides C-optimized math functions (sqrt, trig, log, constants). The `random` module implements the Mersenne Twister PRNG for generating pseudo-random values. "Pseudo-random" means the numbers look random but are generated by a deterministic algorithm — you can reproduce results with `seed()`.

## Code Example

```python
"""Game probability, statistical sampling, and Monte Carlo simulation."""

import math
import random


def dice_probability(target: int, dice: int = 2, sides: int = 6) -> float:
    """Monte Carlo estimate of rolling a target sum with n dice."""
    trials = 100_000
    hits = sum(
        1 for _ in range(trials)
        if sum(random.randint(1, sides) for _ in range(dice)) >= target
    )
    return hits / trials


def sample_statistics(population: list, sample_size: int) -> dict:
    """Draw a random sample and compute statistics."""
    sample = random.sample(population, min(sample_size, len(population)))
    mean = sum(sample) / len(sample)
    variance = sum((x - mean) ** 2 for x in sample) / (len(sample) - 1)
    return {
        "sample_mean": round(mean, 2),
        "sample_std": round(math.sqrt(variance), 2),
        "min": min(sample),
        "max": max(sample),
    }


random.seed(42)
print(f"Probability of rolling 7+ with 2d6: {dice_probability(7):.1%}")
print(f"Stats: {sample_statistics(list(range(1000)), 50)}")
print(f"Pi approx: {math.pi:.4f}")
print(f"Sin(π/2): {math.sin(math.pi / 2)}")
```

## 🔍 How It Works
- `math` wraps C math library — fast, double-precision floating point
- `random` uses Mersenne Twister — good statistical properties, not crypto-secure
- `seed(n)` sets the initial state — use the same seed to reproduce results
- `random.sample(pop, k)` picks k unique items without replacement

## ⚠️ Common Pitfall
Using `random` for security (passwords, tokens). The Mersenne Twister is predictable. Use `secrets` module for cryptography.

## 🧠 Memory Aid
**"math = calculator, random = dice"**: Math gives you exact scientific functions; random gives you chance and simulation.
