## H3 (Hungry Hungry Hippo)

### Architecture
```
x → SSM → gate → SSM → gate → output
```

H3 stacks two SSMs interleaved with gated connections:
1. **First SSM**: Shift operation — provides position-dependent information
2. **Gate**: Elementwise multiplication with input projection
3. **Second SSM**: Content-based addressing (like attention but linear)
4. **Output gate**: Combine with residual

### Key Concepts
- **Shift SSM**: $A$ matrix designed as shift operator (each state stores previous value)
- **Gated SSM**: $B, C$ enable content-based filtering
- **FlashFFTConv**: FFT-based convolution with flash attention-style GPU optimization

## Hyena

### Implicit Long Convolution
Replace attention with data-controlled convolutions:

$$y_t = \sum_{i=0}^{L-1} h_i(x) x_{t-i}$$

- $h_i(x)$: filter weights generated by a small MLP conditioned on position
- No explicit filter bank — implicit parameterization via MLP

### Hyena Operator
$$\text{Hyena}(\{x, q, k, v\})_t = \sum_{i=0}^{L-1} \text{MLP}(\text{pos}_i) \cdot (q_t \cdot k_{t-i}) \cdot v_{t-i}$$

- **Order**: $q$, $k$, $v$ projections, elementwise gating, implicit filter
- **Complexity**: $O(L \log L)$ via FFT-based computation

### Hierarchy
Hyena stacks multiple operators (Hyena-1, Hyena-2, ..., Hyena-N) to achieve hierarchical receptive fields. Each level has a different filter length.

## RWKV (Reinventing RNNs)

### WKV (Weighted Key-Value) Mechanism
$$wkv_t = \frac{\sum_{i=1}^{t-1} \exp(-(t-1-i)w + k_i) \cdot v_i + \exp(u + k_t) \cdot v_t}{\sum_{i=1}^{t-1} \exp(-(t-1-i)w + k_i) + \exp(u + k_t)}$$

- Time-mixing via exponential decay with learned decay $w$
- $u$: bonus for current token
- $k_i, v_i$: key and value projections

### RWKV Block
```
Time mixing: x ← x + WKV(norm(x))
Channel mixing: x ← x + MLP(norm(x))
```

### Training vs Inference
- **Training**: Can be parallelized via cumulative sum (similar to attention parallelism)
- **Inference**: $O(1)$ per token — pure RNN mode

## Comparison Table

| Feature | H3 | Hyena | RWKV | Mamba |
|---------|-----|-------|------|-------|
| Core operation | Stacked SSMs | Implicit convolution | WKV decay | Selective scan |
| Position encoding | Shift SSM | Learned filter pos | Direct decay | Data-dependent |
| Training complexity | $O(L \log L)$ | $O(L \log L)$ | $O(L)$ | $O(L)$ |
| Inference per step | $O(1)$ | $O(1)$ | $O(1)$ | $O(1)$ |
| Long-range | Good (HiPPO) | Excellent (long filter) | Good (exponential decay) | Excellent (selective) |
| Language quality | Good | Good | Good | SOTA |
| Custom CUDA? | FlashFFTConv | FFT (no custom) | No | Yes (selective scan) |

## Code: RWKV Time Mixing

```python
import torch
import torch.nn as nn

class RWKVTimeMixing(nn.Module):
    def __init__(self, d_model, d_state=64):
        super().__init__()
        self.time_decay = nn.Parameter(torch.ones(d_model))
        self.time_first = nn.Parameter(torch.ones(d_model))
        self.key = nn.Linear(d_model, d_state)
        self.value = nn.Linear(d_model, d_model)
        self.receptance = nn.Linear(d_model, d_model)
        self.output = nn.Linear(d_model, d_model)

    def forward(self, x):
        B, T, D = x.shape
        k = self.key(x)
        v = self.value(x)
        r = torch.sigmoid(self.receptance(x))
        
        # WKV computation (simplified for inference)
        wkv = torch.zeros_like(x)
        for t in range(T):
            if t == 0:
                wkv[:, t] = self.time_first * k[:, t] * v[:, t]
            else:
                decay = torch.exp(-self.time_decay)
                wkv[:, t] = (wkv[:, t-1] * decay) + k[:, t] * v[:, t]
        
        return self.output(r * wkv / (torch.abs(wkv).sum(-1, keepdim=True) + 1e-8))
```

## Practical Considerations
- **H3**: Best for tasks requiring content-based recall (copy, lookup)
- **Hyena**: Best when long filter sizes are needed (audio, genomics)
- **RWKV**: Best for low-latency inference (chatbots, streaming)
- **All**: Train with standard AdamW, same learning rate schedule as transformers

## State Space vs Attention

| Property | SSM | Attention |
|----------|-----|-----------|
| Content-based recall | No (H3 partially addresses) | Yes |
| Compressibility | Must compress to state | Can attend to any past token |
| Long sequences | Linear (or $L \log L$) | Quadratic |
| Inductive bias | Markov/continuous | Set-based (unordered) |
