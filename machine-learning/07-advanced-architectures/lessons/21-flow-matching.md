- Understand flow matching for simulation-free continuous normalizing flow training
- Implement conditional flow matching (CFM) loss
- Apply rectified flow for 1-step generation
- Compare flow matching with diffusion and normalizing flows

## Theory
Flow matching provides a simulation-free approach to training continuous normalizing flows by directly regressing a vector field.

### Probability Path
Define a time-dependent probability density $p_t(x)$ connecting $p_0$ (noise) to $p_1$ (data):

$$p_0 = \mathcal{N}(0, I), \quad p_1 = p_{\text{data}}, \quad t \in [0, 1]$$

### Vector Field
The flow is generated by a time-dependent vector field $v_t(x)$:

$$\frac{d}{dt} \psi_t(x) = v_t(\psi_t(x), t), \quad \psi_0(x) = x$$
$$p_t = [\psi_t]_* p_0 \quad \text{(pushforward)}$$

## Conditional Flow Matching (CFM)

### Key Insight
Instead of computing $p_t$ directly (which requires solving ODE), condition on data point $x_1$:

$$p_t(x \mid x_1) = \mathcal{N}((1-t) x_1, \sigma_t^2 I)$$

**Simple path**: $x_t = (1-t) x_0 + t x_1$ where $x_0 \sim \mathcal{N}(0, I)$.

### Vector Field for Conditional Path
$$u_t(x \mid x_1, x_0) = x_1 - x_0$$

### CFM Loss
$$\mathcal{L}_{\text{CFM}} = \mathbb{E}_{t \sim \mathcal{U}[0,1], x_1 \sim p_{\text{data}}, x_0 \sim \mathcal{N}(0, I)} \left[ \|v_\theta(x_t, t) - (x_1 - x_0)\|^2 \right]$$

- **Simulation-free**: No ODE solver needed during training
- **Unbiased gradient**: $\nabla_\theta \mathcal{L}_{\text{CFM}} = \nabla_\theta \mathcal{L}_{\text{FM}}$ (same gradient as true flow matching)

## Rectified Flow

### Straightening
Rectified flow iteratively "straightens" the transport path:

1. Sample pairs $(x_0, x_1)$ from coupling $\pi(x_0, x_1)$
2. Learn velocity $v_\theta$ via CFM loss
3. New coupling: $(x_0, \psi_1(x_0))$ where $\psi_1$ is learned flow

**After $K$ reflows**: Path becomes nearly straight — can generate $x_1$ from $x_0$ in 1-2 steps.

### Key Equation
$$v^* = \arg\min_v \mathbb{E}_{t \sim \mathcal{U}[0,1], (x_0, x_1) \sim \pi} \left[ \|v(x_t, t) - (x_1 - x_0)\|^2 \right]$$

where $x_t = (1-t) x_0 + t x_1$.

## Comparison with Other Methods

| Method | Training | Inference Steps | Path Type |
|--------|----------|----------------|-----------|
| DDPM | Score matching | 100-1000 | Curved (SDE) |
| DDIM | Score matching | 50-100 | Deterministic (curved) |
| CFM | Flow matching | 50-100 | Curved |
| Rectified Flow | Flow matching | 1-10 | Nearly straight |
| Consistency | Consistency loss | 1 | Straight |

## Code: Rectified Flow

```python
import torch
import torch.nn as nn

class RectifiedFlow(nn.Module):
    def __init__(self, backbone):
        super().__init__()
        self.backbone = backbone

    def cfm_loss(self, x1):
        batch = x1.shape[0]
        x0 = torch.randn_like(x1)
        t = torch.rand(batch, device=x1.device)
        
        # Interpolate
        t_expand = t.view(-1, *([1] * (x1.ndim - 1)))
        xt = (1 - t_expand) * x0 + t_expand * x1
        
        # Predict velocity
        v_pred = self.backbone(xt, t)
        target = x1 - x0
        
        return nn.functional.mse_loss(v_pred, target)

    @torch.no_grad()
    def sample(self, num_steps=10, shape=(1, 3, 32, 32)):
        x = torch.randn(*shape, device=next(self.parameters()).device)
        dt = 1.0 / num_steps
        for i in range(num_steps):
            t = torch.full((shape[0],), i * dt, device=x.device)
            v = self.backbone(x, t)
            x = x + v * dt
        return x
```

## Advantages
- **Simulation-free training**: No ODE solver or MCMC needed
- **Fast inference**: Rectified flow enables 1-10 step generation
- **Deterministic mapping**: Same noise always produces same output
- **Straight paths**: Minimize transport curvature, enabling distillation
- **Continuous**: Naturally handles continuous inputs and outputs

## Practical Considerations
- **Time scheduling**: Uniform $t \sim \mathcal{U}[0,1]$ is standard; non-uniform can improve quality
- **Reflow**: One reflow step (rectify) often sufficient; more reflows = straighter paths
- **Distillation**: Train a student model on teacher's $x_0 \to x_1$ mapping for 1-step generation
- **Loss weighting**: Uniform weight works; SNR weighting can improve high-freq details

## Limitations
- **Path quality**: Single CFM path may not be optimal for all data distributions
- **Reflow complexity**: Each reflow requires training another model
- **ODE integration**: Even straight paths have some curvature; exact 1-step can lose quality
- **Modality coverage**: May not fully cover all modes with few steps
