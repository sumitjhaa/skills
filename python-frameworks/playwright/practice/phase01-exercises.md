# 📝 Playwright Practice — Phase 01

## Exercise 1: Multi-Page Navigation

Create a script that navigates through 3 different pages (e.g., example.com, httpbin.org, and a local data URL). Take full-page screenshots of each and save them with distinct filenames.

## Exercise 2: Form Filler

Build a script that fills out a form (use `data:text/html` with inputs for name, email, password, and a select dropdown). Submit the form and verify the URL changed or the form cleared.

## Exercise 3: Waiting for Dynamic Content

Create an HTML page with a button that creates a new `<p>` element after 2 seconds. Use `wait_for_selector` to wait for it, then assert it is visible.

## Exercise 4: Assertion Suite

Write 5 `expect()` assertions for example.com covering: URL, title, element text, element count (links on the page), and that the body contains specific text.

## Exercise 5: Popup Chain

Open a page that opens a popup which then opens another popup. Track all popups and close them in reverse order.

## Exercise 6: API Mocking

Intercept all API calls to `httpbin.org/get` and return a custom JSON response `{"mocked": true}`. Navigate to a page that fetches this endpoint and verify the mocked response.

## Exercise 7: Cookie Manager

Set 3 cookies with different domains. Save storage state. Load it in a new context and verify all 3 cookies are present.

## Exercise 8: Multiple Devices

Test example.com on 4 different device presets (iPhone 13, Pixel 5, iPad Pro, Galaxy S9). For each, print the viewport size and take a screenshot.

## Exercise 9: Visual Regression

Take a screenshot of example.com. Modify the page (add a red border to h1). Take another screenshot. Write a function that compares the two and returns the pixel difference count.

## Exercise 10: Performance Budget

Navigate to example.com and check:
- Page load time < 2000ms
- At least 3 console logs captured
- All network requests succeed (status 200)
Report pass/fail for each.

## Exercise 11: API CRUD

Using the request context, perform a full CRUD cycle against httpbin:
1. POST data
2. GET to verify
3. PUT to update
4. GET to confirm update
5. DELETE

## Exercise 12: Parallel Contexts

Create 4 isolated browser contexts. Navigate each to different URLs. Run them concurrently using `threading` and measure total time vs sequential time.

## Exercise 13: Trace Viewer

Record a trace that includes:
- Navigation to example.com
- A console log injection
- A screenshot
Stop the trace and verify the zip file is non-empty.

## Exercise 14: Custom Report Generator

Write a Python function that reads a JUnit XML (generated by Playwright) and produces a markdown summary with: total tests, passed, failed, and a list of failed test names.

## Exercise 15: End-to-End Workflow

Create a full pipeline that:
1. Launches a device-emulated context (iPhone 13)
2. Navigates to example.com
3. Takes a screenshot with mask on the h1 element
4. Captures performance metrics
5. Saves authentication state
6. Restores it in a new context
7. Generates a trace
8. Produces a JUnit-style summary report

Print all results at the end.
