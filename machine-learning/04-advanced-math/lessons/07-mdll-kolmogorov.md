# 04.07 Minimum Description Length and Kolmogorov Complexity

## Motivation
Occam's razor made formal: the best model is the one that compresses the data most effectively. The Minimum Description Length (MDL) principle and Kolmogorov complexity connect learning, compression, and algorithmic information theory. These ideas provide a rigorous foundation for model selection that avoids overfitting without requiring a separate validation set.

## Learning Objectives
- Define Kolmogorov complexity and understand its incomputability.
- Formulate the MDL principle for model selection.
- Distinguish between two-part and refined MDL.
- Apply MDL to practical problems: tree size selection, clustering, and anomaly detection.

## Math Foundation

### Kolmogorov Complexity
The Kolmogorov complexity $K(x)$ of a binary string $x$ is the length of the shortest program that, when run on a universal Turing machine $U$, outputs $x$ and halts:

$$K_U(x) = \min_{p: U(p) = x} \ell(p)$$

The Invariance Theorem guarantees that the choice of universal Turing machine affects $K$ by at most an additive constant: $|K_U(x) - K_{U'}(x)| \le c_{U,U'}$.

### Properties of Kolmogorov Complexity
1. **Incomputability**: $K(x)$ is not computable (no algorithm can compute $K(x)$ for all $x$). However, it can be *approximated* from above via compression.
2. **Upper bound**: $K(x) \le |x| + O(1)$ (just print $x$ verbatim).
3. **Conditional complexity**: $K(x|y)$ is the length of the shortest program that outputs $x$ given $y$ as input.
4. **Mutual information**: $I(x;y) = K(x) + K(y) - K(x,y)$, analogous to probabilistic mutual information.

### Algorithmic Probability (Solomonoff Prior)
The algorithmic probability $m(x)$ is the probability that a random program (each bit chosen by fair coin flip) outputs $x$:

$$m(x) = \sum_{p: U(p) = x} 2^{-\ell(p)}$$

Up to a multiplicative constant, $m(x) \approx 2^{-K(x)}$. This is the universal prior for Bayesian inference — assigning higher prior probability to simpler strings.

## Minimum Description Length Principle

### Two-Part MDL
Given data $D$ and a set of models $\mathcal{M}$, choose the model $M$ that minimises:

$$L(M) + L(D | M)$$

where $L(M)$ is the code length of the model and $L(D|M)$ is the code length of the data encoded using the model. This is a crisp instantiation of Occam's razor: the total description length trades off model complexity against data fit.

### Connection to Bayesian Inference
For a Bayesian model with prior $P(M)$ and likelihood $P(D|M)$:

$$-\log_2 P(M) - \log_2 P(D|M) = L(M) + L(D|M)$$

The optimal code length for an event with probability $P$ is $-\log_2 P$. Therefore MDL is equivalent to MAP (Maximum a Posteriori) estimation under the optimal code. The difference is epistemological: MDL doesn't require a subjective prior — it uses the "universal prior" based on code length.

### Refined MDL
Two-part MDL has a problem: the division into "model" and "data given model" is arbitrary. Refined MDL uses the normalized maximum likelihood (NML) distribution:

$$p_{\text{NML}}(D) = \frac{p(D|\hat{\theta}(D))}{\int_{\mathcal{D}} p(D'|\hat{\theta}(D')) dD'}$$

The stochastic complexity $-\log p_{\text{NML}}(D)$ gives the optimal code length for data generated by any distribution in the model class. The NML distribution is the unique minimax optimal universal code: it minimises the worst-case regret

$$R(\mathcal{M}) = \min_q \max_{p \in \mathcal{M}} \mathbb{E}_p[-\log q(D) + \log p(D|\hat{\theta}(D))]$$

### Practical MDL Criteria
For linear regression with $k$ parameters and $n$ observations:

$$\text{MDL}(k) = \frac{k}{2} \log n + \frac{n}{2} \log(\text{RSS}/n) + \text{constant}$$

This is identical to the Bayesian Information Criterion (BIC) for Gaussian noise. For decision trees, MDL scores each tree by:
- Code length for the tree structure (splits and thresholds)
- Code length for the class labels given the tree (using empirical frequencies in each leaf)

## Python Implementation

```python
import numpy as np
from sklearn.tree import DecisionTreeClassifier
from scipy.stats import entropy

def mdl_linear_regression(X, y, k):
    """MDL score for linear regression with k parameters.
    Lower is better."""
    n = len(y)
    # OLS estimate
    if k == 1:
        rss = np.sum((y - y.mean())**2)
    else:
        # first k-1 features (plus intercept)
        X_subset = np.column_stack([np.ones(n), X[:, :k-1]])
        beta = np.linalg.lstsq(X_subset, y, rcond=None)[0]
        rss = np.sum((y - X_subset @ beta)**2)
    
    if rss <= 0:
        rss = 1e-12
    
    mdl = (k / 2.0) * np.log(n) + (n / 2.0) * np.log(rss / n)
    return mdl + 0.5 * n * np.log(2 * np.pi) + 0.5 * n

def mdl_decision_tree(X, y, tree):
    """Approximate MDL for a decision tree.
    Code length = structure + threshold values + class labels."""
    n = len(y)
    n_leaves = tree.get_n_leaves()
    n_nodes = tree.tree_.node_count
    
    # Tree structure: encode splits (n_nodes - n_leaves log 3 + n_leaves log 1)
    # Rough approximation: 1 bit per node
    struct_cost = n_nodes * np.log(2)
    
    # Threshold values: assume quantized to sqrt(n) levels
    n_splits = n_nodes - n_leaves
    threshold_cost = n_splits * np.log(np.sqrt(n))
    
    # Class labels in leaves: empirical entropy at each leaf
    leaf_counts = tree.tree_.value[:, 0, :]  # (n_nodes, n_classes)
    label_cost = 0
    for i in range(n_nodes):
        if tree.tree_.children_left[i] == -1:  # leaf
            c = leaf_counts[i]
            p = c / c.sum()
            label_cost += c.sum() * entropy(p, base=np.e)
    
    return struct_cost + threshold_cost + label_cost

def kolmogorov_approximation(string_data):
    """Approximate K(x) using zlib compression length as upper bound."""
    import zlib
    compressed = zlib.compress(string_data.encode())
    return len(compressed), len(string_data) / len(compressed)

# Example: compare linear models with different polynomial degrees
np.random.seed(42)
n = 50
x = np.linspace(-3, 3, n)
y_true = np.sin(x)
y = y_true + np.random.randn(n) * 0.3

X_poly = np.column_stack([x**i for i in range(1, 10)])
mdl_scores = []
for k in range(1, 11):  # intercept + k-1 polynomial terms
    score = mdl_linear_regression(X_poly, y, k)
    mdl_scores.append(score)
    print(f"k={k}: MDL = {score:.2f}")

best_k = np.argmin(mdl_scores) + 1
print(f"Best polynomial degree (MDL): {best_k}")
```

## Connections to Machine Learning

### Model Selection
MDL provides a principled alternative to cross-validation when data is scarce:
- **Clustering**: the optimal number of clusters minimises code length for cluster assignments + residuals.
- **Neural network pruning**: prune connections that increase total description length.
- **Grammar induction**: the best grammar minimises $L(\text{grammar}) + L(\text{data}|\text{grammar})$.

### Anomaly Detection
Normal data can be compressed well (low $K(x)$), while anomalous data has higher Kolmogorov complexity. Approximating $K(x)$ via compression algorithms gives a practical anomaly detector:
- Train a compression model (e.g., gzip, LZMA) on normal data.
- Anomaly score = compressed length of test sample when compressed with the model.

### Variational Autoencoders and the MDL View
The VAE objective decomposes as:

$$\mathbb{E}_{q(z|x)}[-\log p(x|z)] + D_{\text{KL}}(q(z|x) \| p(z))$$

This is exactly a two-part MDL: the KL term $L(z)$ (cost of encoding the latent code) and the reconstruction term $L(x|z)$ (cost of encoding the data given the code). The bits-back argument shows that the VAE actually achieves an MDL-optimal code when the ELBO is tight.

### Connection to Generalisation
The PAC-Bayes bound uses a similar trade-off:

$$L_{\text{test}}(w) \le L_{\text{train}}(w) + \sqrt{\frac{D_{\text{KL}}(Q \| P) + \log(2n/\delta)}{2n-1}}$$

Here $D_{\text{KL}}(Q\|P)$ plays the role of the model complexity (code length), exactly analogous to the model cost in MDL.

## Practical Considerations

### Approximation via Compression
Since $K(x)$ is incomputable, practical MDL uses computable approximations:
- **Gzip/LZMA**: universal Lempel-Ziv compression (asymptotically optimal for stationary ergodic sources).
- **Prediction by Partial Matching (PPM)**: excellent for discrete sequences.
- **Neural compressors**: learned compression with a VAE or autoregressive model — can approach the true $K(x)$ for structured data like images.

### Limitations
1. The optimal code depends on the description language. Two MDL analyses using different code families may yield different conclusions.
2. For finite data, the NML normalising integral may be intractable, requiring approximations (e.g., plug-in codes, Bayesian marginal likelihood).
3. MDL assumes the data is compressible — for truly random data, the shortest description is the data itself, and no model is selected (consistent with the idea that noise cannot be learned).
