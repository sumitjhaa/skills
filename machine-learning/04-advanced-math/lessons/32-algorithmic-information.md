# 04.32 Algorithmic Information Theory

## Motivation
Algorithmic information theory (AIT) quantifies the complexity of individual objects via Kolmogorov complexity — the length of the shortest program that outputs the object. It connects randomness, compression, and inference at a fundamental level. AIT provides the mathematical foundation for the minimum description length principle, Solomonoff induction (optimal universal prediction), and algorithmic approaches to causality.

## Learning Objectives
- Define Kolmogorov complexity and understand its incomputability.
- Apply algorithmic probability and Solomonoff induction to prediction.
- Use compression-based approximations for anomaly detection and model selection.
- Understand the algorithmic information criterion for causal inference.

## Math Foundation

### Kolmogorov Complexity
For a universal Turing machine $U$, the Kolmogorov complexity of a binary string $x$ is:

$$K_U(x) = \min_{p: U(p) = x} \ell(p)$$

where $p$ is a program that outputs $x$ and halts. The **Invariance Theorem** guarantees that for any two universal Turing machines $U$ and $V$:

$$|K_U(x) - K_V(x)| \le c_{U,V}$$

so $K(x)$ is well-defined up to an additive constant.

### Properties
1. **Incomputability**: $K(x)$ is not computable. No algorithm can compute $K(x)$ for all $x$.
2. **Upper bound**: $K(x) \le |x| + O(1)$ (the program "print x").
3. **Lower bound**: $K(x) \ge |x| - c$ for most strings (incompressible = random).
4. **Conditional complexity**: $K(x|y)$ is the length of the shortest program outputting $x$ given input $y$.
5. **Algorithmic mutual information**: $I(x:y) = K(x) + K(y) - K(x,y)$, measuring shared algorithmic information.

### Algorithmic Probability (Solomonoff Prior)
The algorithmic probability of $x$ is the probability that a random program (bits drawn i.i.d. fair coin flips) outputs $x$:

$$m(x) = \sum_{p: U(p) = x} 2^{-\ell(p)}$$

Key properties:
- $m(x) \approx 2^{-K(x)}$ (up to a multiplicative constant).
- $m(x)$ is a universal prior: it dominates any computable semi-measure.
- Solomonoff's universal predictor uses $m$ to predict sequences optimally.

### Solomonoff Induction
Given a sequence $x_{1:t}$, the predictive distribution for $x_{t+1}$ is:

$$P(x_{t+1} | x_{1:t}) = \frac{m(x_{1:t+1})}{m(x_{1:t})}$$

This converges rapidly to the true distribution if the data is generated by a computable process, and no other predictor can do better in terms of total prediction error.

### The Halting Probability $\Omega$
Chaitin's constant $\Omega = \sum_{p: U(p) \text{ halts}} 2^{-|p|}$ is the probability that a random program halts. It is:
- Well-defined (converges).
- Incomputable (knowing $\Omega$ would solve the halting problem).
- Algorithmically random: $K(\Omega_{1:n}) = n - O(1)$.

## Python Implementation

```python
import numpy as np
import zlib

def kolmogorov_approx(data):
    """Approximate K(x) via compression length (upper bound)."""
    if isinstance(data, str):
        data = data.encode()
    compressed = zlib.compress(data)
    return len(compressed), len(compressed) / len(data)

def algorithmic_anomaly_score(data, reference_data):
    """Anomaly detection via compression.
    Score = (compressed length of reference + data - compressed length of reference)"""
    if isinstance(data, str):
        data = data.encode()
    if isinstance(reference_data, str):
        reference_data = reference_data.encode()
    
    # Train a compression model on reference data
    compressor = zlib.compressobj()
    compressor.compress(reference_data)
    compressor.flush()
    
    # Compression-based anomaly score
    common = zlib.compress(reference_data + b'\n' + data)
    ref_only = zlib.compress(reference_data)
    
    # The score is the additional bits needed to encode data given reference
    score = len(common) - len(ref_only)
    return score

def algorithmic_causality(X, Y):
    """Algorithmic information criterion for causal direction.
    Lower K(X) + K(Y|X*) vs K(Y) + K(X|Y*) suggests X -> Y."""
    # Compression-based approximation
    x_str = str(X.tobytes())
    y_str = str(Y.tobytes())
    
    K_x = len(zlib.compress(x_str.encode()))
    K_y = len(zlib.compress(y_str.encode()))
    K_xy = len(zlib.compress((x_str + y_str).encode()))
    
    I = K_x + K_y - K_xy  # algorithmic mutual information
    
    # Causal direction via minimum description length
    mdl_forward = K_x + len(zlib.compress((y_str + x_str).encode())) - K_x  # K(Y|X*)
    mdl_backward = K_y + len(zlib.compress((x_str + y_str).encode())) - K_y  # K(X|Y*)
    
    return {
        'mutual_information': I,
        'causal_strength_forward': mdl_forward,
        'causal_strength_backward': mdl_backward,
        'preferred_direction': 'X->Y' if mdl_forward < mdl_backward else 'Y->X'
    }

# Example: anomaly detection
normal_data = "hello world this is normal data " * 100
anomalous_data = "zzz xyzzy abcdefghijk lmnop " * 100

normal_score = algorithmic_anomaly_score(normal_data[:100], normal_data[:1000])
anomaly_score = algorithmic_anomaly_score(anomalous_data[:100], normal_data[:1000])
print(f"Normal sample anomaly score: {normal_score}")
print(f"Anomalous sample anomaly score: {anomaly_score}")

# Example: incompressible vs compressible
random_bits = ''.join(np.random.choice(['0', '1'], size=1000))
zeros = '0' * 1000
print(f"Random string compression ratio: {kolmogorov_approx(random_bits)[1]:.3f}")
print(f"Zeros string compression ratio: {kolmogorov_approx(zeros)[1]:.3f}")
```

## Connections to Machine Learning

### MDL for Model Selection
The MDL principle selects the model minimising $L(M) + L(D|M)$, where $L(M)$ approximates $- \log P(M)$ and $L(D|M)$ is the code length of data under the model. For neural networks, the description length includes:
- Network architecture (structure).
- Parameter quantisation (weights at finite precision).
- Residuals (prediction errors under the model).

### Anomaly Detection via Compression
Simple principle: normal data can be compressed well, anomalies cannot. Practical methods:
- Train a compression model (LZMA, gzip, or a learned neural compressor) on normal data.
- For a test sample, compute the compressed size difference: $K(x \cup D) - K(D)$.
- Large difference $\implies$ anomalous.

This works without labels and adapts to the data distribution.

### Algorithmic Information for Causality
The algorithmic Markov condition states that if $X \to Y$, then $K(P(X,Y)) = K(P(X)) + K(P(Y|X))$ (up to an additive constant). The causal direction minimises:

$$K(P_{\text{cause}}) + K(P_{\text{effect|cause}}) < K(P_{\text{effect}}) + K(P_{\text{cause|effect}})$$

In practice, this is approximated using compression algorithms or kernel methods. Applications include distinguishing cause and effect in observational data.

### Neural Network Generalisation as Compression
Zhang et al. (2017) showed neural networks can memorise random labels, but they generalise only when the labels are structured. The compression view: a network that generalises has a short description length of the weights (small $L(M)$) given the training data. PAC-Bayes bounds connect the KL divergence between the posterior and prior to description length.

## Practical Considerations

### Approximating Kolmogorov Complexity
Since $K(x)$ is incomputable, all practical methods use upper bounds:
- **LZ77/LZMA**: $O(n)$ compression, good for local patterns.
- **PPM (Prediction by Partial Matching)**: $O(n^2)$, better for text.
- **Neural compressors**: $O(n)$ once trained, best compression ratios but require training data.

### When to Use Algorithmic Information
- **Small data**: AIT excels when data is too scarce for statistical methods.
- **Model selection**: when comparing models of different complexities (tree size, number of clusters).
- **Causal discovery**: when interventions are impossible.
- **Anomaly detection**: in high-dimensional, unlabelled settings.
