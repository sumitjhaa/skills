# 📘 Django Phase 04 — Lesson 05: Routers

> 🎯 **Goal**: Use DRF routers to auto-generate URL patterns from ViewSets.

## 📖 Concepts

### Why Routers?
Without a router, you write URLs manually:
```python
urlpatterns = [
    path('posts/', PostViewSet.as_view({'GET': 'list', 'POST': 'create'})),
    path('posts/<int:pk>/', PostViewSet.as_view({
        'GET': 'retrieve', 'PUT': 'update', 'PATCH': 'partial_update', 'DELETE': 'destroy'
    })),
]
```

With a router:
```python
router = DefaultRouter()
router.register('posts', PostViewSet, basename='post')
urlpatterns = router.urls
```

### Router Types

| Router | Features |
|--------|----------|
| `DefaultRouter` | All routes + API root view + trailing slash |
| `SimpleRouter` | All routes, no API root |

### Generated URLs

| URL Pattern | Method | Action |
|-------------|--------|--------|
| `/posts/` | GET | list |
| `/posts/` | POST | create |
| `/posts/{pk}/` | GET | retrieve |
| `/posts/{pk}/` | PUT | update |
| `/posts/{pk}/` | PATCH | partial_update |
| `/posts/{pk}/` | DELETE | destroy |
| `/posts/{pk}/custom/` | any | @action(detail=True) |
| `/posts/custom/` | any | @action(detail=False) |

### Basename
- If you have a `queryset`, DRF infers the basename from the model
- If no `queryset`, you MUST provide `basename`
- Basename determines URL name: `post-list`, `post-detail`

### ADHD-Friendly Summary
```
router = DefaultRouter()
router.register('posts', PostViewSet)
urlpatterns = router.urls
→ auto-generates list + detail + custom action URLs
```

## 🛠️ Code

```python
# urls.py
from django.urls import path, include
from rest_framework.routers import DefaultRouter
from .views import PostViewSet, CommentViewSet

router = DefaultRouter()
router.register('posts', PostViewSet, basename='post')
router.register('comments', CommentViewSet, basename='comment')

urlpatterns = [
    path('api/', include(router.urls)),
]

# Equivalent manual URLs (generated by router):
# /api/posts/              GET list, POST create
# /api/posts/{pk}/         GET retrieve, PUT update, PATCH partial_update, DELETE destroy
# /api/posts/{pk}/publish/ POST (custom action)
# /api/comments/           GET list, POST create
# /api/comments/{pk}/      GET retrieve, PUT update, PATCH partial_update, DELETE destroy
```

## 🧪 Practice

2. Check `router.urls` — list the generated patterns
3. Add a second `AuthorViewSet` and register it
4. Use `router.register` with `basename` when no queryset
5. Use `SimpleRouter` and compare the URL list

## 🧠 Key Takeaways

- Routers eliminate manual URL-view mapping for ViewSets
- `DefaultRouter` adds an API root view at `/`
- `basename` is auto-detected from queryset model
- Custom `@action` routes are auto-registered
- Use `include(router.urls)` in your main `urlpatterns`
